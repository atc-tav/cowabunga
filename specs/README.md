# Game Specs

Reference design specifications for arcade-game clones, authored by an external
consultant. These are **source material / research** — the authoritative,
detailed description of each game's mechanics, scoring, sprites, and constants
to build (or validate) a faithful clone.

> These specs are reference inputs, not a description of what's currently built.
> Where a spec and the live implementation differ, the spec is the *target* to
> reason from — not a guarantee of current behavior. Build/architecture
> conventions still live in the root `CLAUDE.md`; product strategy lives in
> `docs/`.

> **The spec is raw material, not a finished game — and not even the hard part.**
> All four of our audited clones had rich specs; three still came out fake. A
> spec only becomes a faithful game when someone converts it into an **Oracle
> Ledger** and drives the implementation to green against it. Read the process
> first: [`docs/spec-driven-development.md`](../docs/spec-driven-development.md).
> Before building *from* a spec, lint it against
> [`SPEC_GUIDE.md`](./SPEC_GUIDE.md) (or run `/spec-lint <game>`) — several of
> these specs contain silent internal contradictions that would be faithfully
> implemented as bugs.

**Notes**
- The **NES Dig Dug** spec is a *delta* on top of the **arcade Dig Dug** spec —
  read the arcade one first for shared mechanics, then the NES one for the
  differences (flagged `⚠️ NES DIFF:`).
- Each spec embeds `> ✅ CHECK:` callouts — self-verification notes intended to
  validate an implementation against the spec.

## Adding a spec

Drop the consultant's markdown here with a clear, lowercase-kebab filename
(e.g. `pac-man-arcade.md`) and add a row to the index table above. New specs
must clear the bar in [`SPEC_GUIDE.md`](./SPEC_GUIDE.md) — that doc defines what
a buildable spec contains and the lint checklist it has to pass.

> **Pac-Man has no spec** (it predates this folder), so it cannot be scored for
> faithfulness or regenerated by the factory. Back-filling `pac-man-arcade.md`
> to the `SPEC_GUIDE.md` bar is the prerequisite to holding it to the Arkanoid
> standard.
